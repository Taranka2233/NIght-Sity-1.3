# Найт-Сити Нет

Android-мессенджер в киберпанк-стиле. Клиент работает в Capacitor 8, данные и авторизация — Firebase, входящие звонки — через FCM и Cloudflare Worker.

Текущая версия приложения: **2.077.209**.

## Что находится в репозитории

- `index.html` — полный клиент приложения;
- `src/firebase-bridge.mjs` — локально собираемый Firebase SDK;
- `firestore.rules`, `storage.rules` — правила доступа;
- `worker.js` — защищённый шлюз FCM для звонков;
- `android/` — воспроизводимый проект Capacitor 8;
- `setup_icons.py`, `setup_calls.py` — идемпотентная Android-настройка;
- `test/` и `scripts/` — проверки правил и целостности сборки.

## Локальная проверка

Требуются Node.js 22+ и JDK 21.

```bash
npm ci
npm run verify
npm audit --audit-level=low
npm run test:rules
npm run android:debug
```

APK появится в `android/app/build/outputs/apk/debug/app-debug.apk`. Такой APK подписан временным отладочным ключом и подходит для тестирования. Для публикации нужна отдельная release-подпись, хранящаяся вне Git.

## Безопасность

- Личные и групповые сообщения отправляются только после успешного E2E-шифрования; переход к открытому тексту запрещён.
- Публичный профиль не содержит email, телефона и FCM-токена.
- Worker принимает Firebase ID-токен, проверяет чат и звонок и сам получает приватный FCM-токен адресата.
- Firestore запрещает подмену отправителя, чтение чужих приватных документов и сигналинг посторонними.
- `storage.rules` пока полностью запрещает Firebase Storage: это безопасный режим до реализации шифрования файлов перед загрузкой.

Подробный порядок развёртывания и обязательные секреты описаны в `ДЕПЛОЙ_И_ОПИСАНИЕ.md`, результаты аудита — в `AUDIT.md`.
