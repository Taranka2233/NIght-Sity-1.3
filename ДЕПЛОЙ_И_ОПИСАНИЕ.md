# Найт-Сити Нет — финальная сводка, файлы и деплой

Мессенджер в стилистике Cyberpunk 2077: E2E-шифрование (ECDH P-256 + AES-GCM), Firebase (Spark), Cloudflare Worker для пуш-звонков, упаковка в Android APK через Capacitor 6 + GitHub Actions. Репозиторий: `Taranka2233/Night-Sity-1.2`.

---

## 1. Файлы и куда что деплоить

Четыре разных места. Не перепутай — половина проблем была из-за вставки файла не туда.

| Файл | Куда | Как |
|------|------|-----|
| **index.html** | GitHub-репо (корень) | заменить файл → commit в `main` → пересборка APK |
| **build-apk.yml** | GitHub-репо → `.github/workflows/` | заменить файл → commit |
| **setup_calls.py** | GitHub-репо (корень) | уже залит; трогать не надо |
| **firestore.rules** | Firebase Console → Firestore → вкладка **Rules** | вставить весь текст → **Publish** (в репо НЕ класть) |
| **worker.js** | Cloudflare → Workers & Pages → `nightcity-push` → Edit code | заменить весь скрипт → **Deploy** (в репо НЕ класть) |

Firestore Rules — это **не** JavaScript, начинается с `rules_version = '2';`. Worker — это JavaScript, идёт в Cloudflare. Это два разных файла в двух разных сервисах.

---

## 2. Порядок деплоя (важен)

1. **Firebase Rules** — опубликуй `firestore.rules`. Без них: звонки, приём приглашений в банду и приватный `fcmToken` работать не будут.
2. **Cloudflare Worker** — задеплой `worker.js`. Проверь, что у сервисного аккаунта (`FB_CLIENT_EMAIL`) есть доступ на чтение Firestore (роль Editor). Переменные `FB_PRIVATE_KEY` / `FB_CLIENT_EMAIL` / `FB_PROJECT_ID` — те же.
3. **GitHub** — залей `index.html` и `.github/workflows/build-apk.yml`, сделай commit → дождись сборки APK.
4. **Поставь свежий APK** (переустанови). Разрешения из манифеста применяются только при новой сборке.

## 3. Что сделано — полный список

### Криптография (E2E)
- **C1 — fail-closed.** Раньше при отсутствии ключа собеседника / выключенном e2e сообщение уходило **открытым текстом** под фейковым 🔒. Теперь в личных и групповых чатах чувствительный контент без шифрования **не отправляется** (тост «у собеседника ещё нет ключа»).
- **C2 — шифрование метаполей.** Подписи к фото/видео, имена файлов и текст цитаты-ответа теперь шифруются (раньше лежали открыто даже у «зашифрованных» сообщений). Обратная совместимость сохранена.
- **C3.** Предупреждение, если в банде у части участников нет ключа (они не прочитают сообщение).

### Правила Firestore
- **#1/#2 — звонки.** Раньше сигналинг 1:1 и групповых звонков был открыт любому авторизованному (деанон, MITM, DoS). Теперь — только участникам.
- **#3/#4 — сообщения.** Нельзя создать сообщение от чужого имени (`from == uid`); редактировать/удалять — только автору; реакции (поле `reactions`) — любому участнику.
- **#5 — хэндлы.** Занять `@имя` можно, только записав туда свой `uid`.
- **#6 — чаты.** Удалять участников и менять `owner`/`admins` может только владелец/админ (защита от захвата банды).
- **#7 — приватность.** `email` больше не хранится в публичном профиле (берётся из Firebase Auth), `fcmToken` перенесён в приватную подколлекцию `users/{uid}/private/push`. Старые записи затираются при следующем входе.

### Пуш-звонки (Cloudflare Worker, режим B)
- Раньше эндпоинт был **полностью открыт** — любой мог отправить кому угодно поддельный «входящий звонок» с любым именем и жечь квоту.
- Теперь Воркер проверяет **Firebase ID-токен** отправителя (подпись по публичным ключам Google), сам достаёт токен получателя из Firestore (клиент его больше не передаёт), проверяет, что оба в одном чате, кэширует access-токен и не отдаёт наружу тело ответа FCM.

### Сборка / Android
- **google-services.json** — есть в репо, FCM настраивается. ✓
- **setup_calls.py** — залит; нативный полноэкранный входящий звонок собирается.
- **B2 — авто-приём звонка.** Нативная кнопка «Принять» (deep-link `nightcity://accept`) теперь реально принимает звонок, без второго подтверждения внутри.
- **Разрешения в манифесте.** Добавлены `READ_CONTACTS`, `ACCESS_FINE/COARSE_LOCATION`, `POST_NOTIFICATIONS`, `READ_MEDIA_IMAGES/VIDEO`, `READ_EXTERNAL_STORAGE`. Без этого Android отклонял запросы без показа диалога — контакты/гео/файлы «не работали».

### Логика и качество
- **Время сообщений.** Раньше все сообщения показывали текущее время (баг: `time = now()`). Теперь — реальное время отправки.
- **Синхронизация статусов разрешений.** Функция `_syncPerms` не вызывалась — статусы на экране разрешений врали. Теперь вызывается при старте.
- Проверены утечки слушателей (нет), гонки/дубли звонков (защищены), секреты/логи (чисто), КИБЕРДЕК 2048 (логика слияний, game-over, речь — без багов).

### Функции
- **Удаление «у себя» / «у всех».** «У себя» прячет сообщение локально (любое, в т.ч. чужое; не всплывает после ресинка). «У всех» стирает с сервера — только для своих (что и разрешают правила).
- **Согласие на вступление в банду (#8).** Приглашённые попадают в `pending`, видят «⚡ приглашение в банду», по тапу — «ВСТУПИТЬ / ОТКЛОНИТЬ». До принятия не видят сообщения банды. Форс-добавление убрано.
- **Запрос разрешений.** «Память/файлы» теперь реально запрашиваются (была заглушка). При первом входе однократно спрашиваются контакты, геолокация и файлы.

## 4. Чек-лист тестирования (нужны два аккаунта)

**Крипто/сообщения**
- Собеседник без ключа → отправка не проходит, тост. С ключами → фото с подписью читается, в базе `caption` нет открытым.
- Старый чат → сообщения показывают **время отправки**, не текущее.

**Удаление**
- Своё сообщение → есть «У СЕБЯ» и «У ВСЕХ»; «У ВСЕХ» → исчезает у обоих.
- Чужое → только «У СЕБЯ»; прячется у тебя, у собеседника остаётся; после перезахода не возвращается.

**Банды (#8)**
- A создаёт банду с B → у B «приглашение в банду», у A — банда.
- B: «ВСТУПИТЬ» → банда появляется; «ОТКЛОНИТЬ» → исчезает. До принятия B не видит сообщений.

**Звонки**
- Заблокированный экран → звонок → «Принять» → приложение открывается и **сразу** соединяет.
- Пустой POST на Воркер (без idToken) → отклоняется, пуш не уходит.

**Разрешения** (после новой сборки APK)
- Настройки → Разрешения → «Контакты» / «Местоположение» / «Уведомления» → появляется системный диалог.
- «Память/файлы»: Android ≤12 — диалог хранилища; 13+ — доступ к медиа.
- Уже выданные разрешения показываются как «РАЗРЕШЕНО».

## 5. Осталось / известные нюансы

- **Память/файлы на Android 13+:** системный диалог классического хранилища не появляется (scoped storage) — это нормально, сохранение работает через приложенческое хранилище и «Поделиться». Диалог медиа появляется, если приложение читает галерею.
- **Состав банды:** имена приглашённых (ещё не принявших) могут показываться в списке участников, хотя счётчик считает только реальных. Косметика.
- **Слишком большое видео в E2E-чате:** системная плашка «видео не отправлено» блокируется fail-closed — редкий безопасный edge.
- **Список «спрятанных у себя»** (`nc_hidden`) со временем растёт — можно позже добавить обрезку.
- **Дедуп/rate-limit пуш-звонков** на Воркере — не делали (низкий приоритет).
- Если после пересборки конкретное разрешение всё ещё не спрашивает — сообщи модель/версию Android, докрутим точечно.
