# ТЕХЗАДАНИЕ / ПРОМПТ: Создание приложения «НАЙТ-СИТИ НЕТ»

> **Как использовать этот документ.** Это полная спецификация зрелого приложения. Из-за масштаба один AI-ответ не построит его целиком — стройте **по модулям** (сначала каркас и дизайн-систему, потом чаты, потом шифрование, звонки и т.д.). Отдавайте AI по одному разделу за раз. Документ также служит «источником правды» о проекте.

---

## 1. КОНЦЕПЦИЯ

**«Найт-Сити Нет» (Night City Net)** — защищённый мессенджер в эстетике **Cyberpunk 2077**. Не просто чат, а иммерсивный киберпанк-интерфейс: неон, глитчи, матрица-дождь, хакерская лексика, встроенные мини-игры. Ощущение — будто общаешься через терминал в Найт-Сити.

**Ключевая идея:** серьёзный функционал (E2E-шифрование, звонки, группы) в обёртке атмосферной киберпанк-игры.

**Аудитория:** энтузиасты киберпанка, приватность-ориентированные пользователи, друзья автора.

## 2. ПЛАТФОРМА И СТЕК

- **Формат:** одностраничное приложение — **один большой HTML-файл** со встроенными CSS и JS.
- **Движок рендера:** кастомный реактивный движок **dc-runtime** (похож на React, но свой). Синтаксис:
  - Экраны: `<sc-if value="{{ isX }}">...</sc-if>`
  - Циклы: `<sc-for list="{{ items }}" as="c">...</sc-for>`
  - Обработчики: `onclick="{{ handler }}"`, `oninput="{{ handler }}"`
  - **Правило движка:** каждый `{{ handler }}` из разметки ОБЯЗАН быть ключом в объекте экспорта рендера (иначе кнопка «мёртвая»). Свойства элементов циклов (`c.tap`) задаются при маппинге списка.
  - Компонент — класс с `state = {}`, методами-стрелками, и методом рендера, возвращающим объект данных для шаблона.
- **Упаковка в Android:** **Capacitor 6** → APK.
- **CI/CD:** **GitHub Actions** — по коммиту собирает APK автоматически.
- **Плагины Capacitor:** push-notifications, local-notifications, camera, filesystem, geolocation, share, contacts, native-biometric, status-bar, app, native-settings.
- **Целевая ОС:** Android (тест на Xiaomi/MIUI + Samsung).

## 4. ФУНКЦИИ (полный список)

### 4.1 Авторизация и аккаунт
- Регистрация/вход по email + паролю (Firebase Auth).
- Уникальный хэндл `@ник` (коллекция handles).
- Профиль: имя, статус, аватар, псевдонимы контактов (локальные).
- Восстановление сессии, **умный офлайн-вход** (если входил раньше — заходит без экрана входа, даже офлайн).
- Локальная блокировка приложения: PIN + биометрия (native-biometric).
- Двухфакторная защита (локальная).
- Заставка-загрузка при каждом запуске (мин. ~1.6с).

### 4.2 Чаты (личные)
- Текст, эмодзи (кастомные SVG-эмодзи), стикеры.
- Медиа: фото, видео, голосовые (запись через MediaRecorder), файлы.
- Ответы (reply), редактирование, удаление, пересылка.
- Реакции на сообщения.
- Закрепление сообщения.
- **Закладки** (сохранённые сообщения, локально).
- Множественный выбор сообщений (панель: переслать/копировать/удалить/в закладки).
- Статусы доставки/прочтения (✓✓).
- Обои чата (тьма/сетка/багровый/синт/схема/свой).
- Псевдонимы контактов («видишь только ты»).

### 4.3 Группы (банды) и каналы
- Создание группы с участниками, аватар, тип.
- Групповые сообщения.
- Каналы (односторонние, с просмотрами).
- Отличия прав от личных чатов.

### 4.4 Звонки (WebRTC)
- Аудио и видеозвонки (getUserMedia + RTCPeerConnection).
- Сигналинг через Firestore (offer/answer/ICE-кандидаты).
- Приём/отклонение, завершение, отмена.
- Свап окон (большое/маленькое) в видеозвонке.
- Групповые звонки (mesh WebRTC).
- **Пуш-уведомления о звонке** (будят телефон) — см. backend.
- **Полноэкранный входящий звонок над блокировкой** (native Java: FirebaseMessagingService + Activity с showWhenLocked). Опционально, сложно на MIUI.

### 4.5 E2E-шифрование
- ECDH P-256 (обмен ключами) + AES-GCM (шифрование).
- Шифруются текст, медиа, групповые сообщения.
- Ключ хранится локально; **резервная копия ключа под паролем** (в Firebase) + восстановление.
- Сервер (Firebase) НЕ видит открытый текст.
- Экранирование HTML в сообщениях (защита от XSS).

### 4.6 Встроенные игры
- **КИБЕРДЕК 2048** — киберпанк-2048 (плитки = программы: BYTE→DATA→...→MATRIX), звания, TOP-рекорды, пиксельный новостной эфир, TTS-озвучка.
- **НЕТРАННЕР** — idle-кликер (взлом → эддисы €$, авто-майнеры, апгрейды, престиж-«флэтлайн», ранги, частицы/комбо/эффекты).
- Игры доступны из Настроек, сохранение в localStorage.

### 4.7 Прочее
- Поиск людей по хэндлу.
- Уведомления (local + push).
- Анимированная шапка (глитч заголовка, статус-точка, сканлайн, циклическая подпись).
- Индикатор офлайна.
- Настройки: тема, обои, звук, иконка приложения (динамические иконки), игры, закладки.

## 5. BACKEND

### 5.1 Firebase (бесплатный тариф Spark)
- **Auth** — email/пароль.
- **Firestore** — коллекции: `users` (профили + fcmToken), `handles` (@ники), `chats/{id}/messages`, `calls` (сигналинг), `groupcalls`.
- **Cloud Messaging (FCM)** — пуши.
- **Правила безопасности** — доступ только авторизованным, к своим данным (см. firestore.rules проекта).
- ⚠️ **Важно:** только Spark (нет Blaze/Cloud Functions — ограничение по региону/карте). Поэтому пуши шлёт внешний сервер.

### 5.2 Cloudflare Worker (сервер пушей)
- Бесплатный воркер, шлёт высокоприоритетный **data-only FCM** пуш при звонке (будит заблокированный телефон).
- Аутентификация в FCM через сервисный аккаунт Firebase (JWT RS256 → OAuth2).
- Секреты (project_id, client_email, private_key) — в переменных Cloudflare, НЕ в коде.
- Приложение при звонке читает fcmToken собеседника из Firestore и POST-ит воркеру.

## 6. СБОРКА (GitHub Actions)

Пайплайн `.github/workflows/build-apk.yml`:
1. Checkout, Node 20, JDK 17.
2. `npm install`, копирование `index.html` → `www/`.
3. `npx cap add android`.
4. **Настройка FCM:** кладёт `google-services.json` в android/app, добавляет google-services в gradle.
5. Разрешения манифеста: RECORD_AUDIO, CAMERA, POST_NOTIFICATIONS.
6. Динамические иконки (`setup_icons.py`).
7. **Native полноэкранный звонок** (`setup_calls.py` — генерит Java, патчит манифест + gradle firebase-messaging).
8. `npx cap sync`, фиксированный debug.keystore (обновление поверх), `gradlew assembleDebug`.
9. Загрузка APK-артефакта.

**Особенность сборки index.html:** ресурсы (эмодзи, обои, иконки, игры) вшиваются как base64 в `<head>` из исходного `template_fixed.html`.

## 7. ОГРАНИЧЕНИЯ И РЕШЕНИЯ (контекст)

- **Регион РФ, нет карты** → Firebase только Spark, нет Cloud Functions → пуши через Cloudflare.
- **Нет TURN-сервера** (нужен Blaze) → видеозвонок между разными сетями может не соединиться; в одной сети/WiFi работает.
- **Гибридный подход:** основное на HTML/JS (WebView), системные фичи (звонок над блокировкой) — точечными native-вставками через Capacitor. Полный перенос на нативный Kotlin/Flutter НЕ нужен на текущем масштабе.
- **MIUI/Xiaomi** — самый капризный OEM для фоновых пушей и звонков (нужны ручные разрешения: автозапуск, показ на блокировке, отключение экономии батареи).

## 8. ПОРЯДОК ПОСТРОЙКИ (рекомендация)

1. **Каркас + дизайн-система** (палитра, шрифты, темы, базовые экраны, движок рендера или React).
2. **Экраны без backend:** заставка, вход/регистрация, список чатов (заглушки), настройки.
3. **Firebase:** Auth + Firestore, живые чаты (текст) между устройствами.
4. **Медиа:** фото/видео/голос/файлы.
5. **E2E-шифрование** поверх чатов.
6. **Группы/каналы.**
7. **Звонки WebRTC** (аудио → видео → групповые).
8. **Пуши** (FCM + Cloudflare Worker).
9. **Native полноэкранный звонок** (по желанию).
10. **Игры** (2048, кликер).
11. **Полировка:** анимации, глитчи, матрица, эффекты, офлайн-режим.

## 9. ЧТО ДЕЛАЕТ ПРИЛОЖЕНИЕ ОСОБЕННЫМ
- Полное E2E-шифрование на бесплатном стеке (без своего сервера).
- Богатая киберпанк-эстетика (не «тема», а цельный мир).
- Встроенные игры уровня отдельных приложений.
- Пуш-звонки без платного Firebase (через Cloudflare).
- Работает офлайн (кэш, локальная сессия).
