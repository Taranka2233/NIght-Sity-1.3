## Оставшиеся ограничения

1. Нет TURN-сервера. Звонки за некоторыми NAT и между разными мобильными сетями могут не соединяться.
2. E2E-вложения ограничены размером документа Firestore: примерно 700 КБ до шифрования/накладных расходов. Storage закрыт до реализации шифрования файлов на клиенте.
3. Часть расширенных переключателей приватности хранится локально и не является серверной политикой доступа.
4. Самоуничтожение скрывает просроченное сообщение у клиентов, но гарантированное серверное удаление требует TTL/backend-задачи, если устройство отправителя офлайн.
5. Для обновляемой production-установки нужен постоянный release-keystore в секретах CI. Debug APK предназначен только для тестирования.
6. Полный приёмочный тест Firebase/FCM/WebRTC требует развёрнутых правил и Worker, двух аккаунтов и двух физических устройств.

Зелёный GitHub Actions run получен. До production-релиза остаётся обязательной проверка пункта 6; остальные ограничения не маскируются интерфейсом как завершённые серверные функции.
